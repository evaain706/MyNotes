

### 1. 선언형 UI

> UI를 “어떻게 변경할지”가 아니라 “어떤 상태일 때 어떤 UI인지”만 정의

- DOM 직접 조작 제거
    
- 상태 → UI 자동 동기화
    
- 예측 가능성 증가
    

---

### 2. 컴포넌트 기반 설계

- UI를 작은 단위로 분리
    
- 관심사 분리 (Separation of Concerns)
    
- 유지보수성 향상
    
- 재사용성 증가
    

---

### 3. 단방향 데이터 흐름

- 데이터는 위 → 아래로 흐름
    
- 상태 추적이 명확
    
- 디버깅 용이
    

---

### 4. Virtual DOM

- 변경 사항 diff
    
- 실제 DOM 최소 조작
    
- 성능 최적화
    

※ 단, Virtual DOM이 React의 본질은 아님  
→ 핵심은 “상태 기반 렌더링 모델”

---

### 5. 상태 관리 패러다임

- UI = f(state)
    
- 상태가 바뀌면 UI 재계산
    
- 예측 가능한 렌더 사이클
