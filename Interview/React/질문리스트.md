## React란?

- **UI를 선언적으로 구성하기 위한 JavaScript 라이브러리**
- SPA에서 **상태 변화에 따른 UI 업데이트를 효율적으로 처리**하기 위해 설계됨
- Facebook(Meta)에서 개발

핵심 목적  
→ **상태(state)가 바뀌면 UI는 자동으로 다시 계산된다

---

## React의 원리 특징 장단점

### 원리

- UI = f(state)

- 상태 변경 → Virtual DOM 재계산 → 실제 DOM 최소 변경

### 특징

- 컴포넌트 기반 구조
- 단방향 데이터 흐름
- 선언형 UI
- Virtual DOM

### 장점

- UI 상태 관리가 명확
- 재사용 가능한 컴포넌트
- 대규모 UI 유지보수에 유리
- 생태계가 매우 큼

### 단점

- 상태 관리 구조를 잘못 잡으면 복잡도 급증
- 러닝커브 (Hooks, 상태 관리, 렌더링 최적화)
- SEO 이슈 (CSR 기준 → SSR로 보완)


---


## V-DOM이란?

- **실제 DOM의 가벼운 JavaScript 객체 표현**

- 메모리 상에서 UI 구조를 표현한 트리


1. 상태 변경
2. 새로운 Virtual DOM 생성
3. 이전 Virtual DOM과 비교(diffing)
4. 변경된 부분만 실제 DOM에 반영(commit)


왜 필요한가?

- 실제 DOM 조작은 매우 비쌈

- 변경 최소화로 성능 확보


--- 

## JSX란?

- JavaScript 안에서 HTML처럼 UI를 작성하는 문법

- **문법 설탕(Syntax Sugar)**

- JS 표현식 사용 가능

- 조건부 렌더링, 반복 렌더링에 최적

---

## 리액트 라이프사이클

컴포넌트의 **생성 → 업데이트 → 제거** 흐름

### 단계

1. Mount
2. Update
3. Unmount

- componentDidMount
- componentDidUpdate
- componentWillUnmount

## 함수형 컴포넌트의 장점과 클래스형 컴포넌트를 안쓰는이유

### 함수형 장점

- 코드가 간결
- Hooks로 로직 재사용 가능
- this 바인딩 문제 없음
- React 팀이 공식적으로 권장


### 클래스형을 안 쓰는 이유

- 로직 재사용이 어려움
- 코드 가독성 저하
- 라이프사이클 메소드 분산
- 신규 기능 지원 우선순위 낮음


**실무 기준: 함수형 컴포넌트 + Hooks가 표준**

---


## React Hooks란?


- 함수형 컴포넌트에서 상태와 생명주기를 사용할 수 있게 해주는 API


대표 Hooks

- useState
- useEffect

---
## State 관리법

- Local State: useState

- Derived State: 계산으로 해결 (state로 만들지 않음)

- Global State:
    
    - Context API
        
    - Redux / Zustand / Jotai
        
    - Server State: React Query
        

**확실한 개선 포인트**

> 서버 데이터는 전역 상태로 관리하지 말고 React Query 사용


---

## State란?

- 컴포넌트의 **변경 가능한 데이터**
- state 변경 → 렌더링 발생


잘못된 예

- state를 직접 변경
    
- 불필요한 state 남발

---


## Props란?

- 부모 → 자식으로 전달되는 **읽기 전용 데이터**

- 컴포넌트 재사용의 핵심


---


## Key Props를 사용하는이유

- 리스트 렌더링 시 **각 요소의 고유성 식별**

- diffing 최적화


**❌ index 사용 문제**

- 요소 순서 변경 시 잘못된 재사용 발생

✅ 안정적인 고유 id 사용 권장


---

## Ref의 용도

- DOM 직접 접근
- 렌더링과 무관한 값 저장


대표 사용처

- input focus
- scroll 위치
- 타이머 id 저장

---



## 제어 컴포넌트와 비제어 컴포넌트의 차이

### 제어 컴포넌트

- value를 state로 관리
- React가 입력 상태를 제어

### 비제어 컴포넌트

- DOM이 상태 관리
- ref로 값 접근


실무 기준

- **폼은 제어 컴포넌트가 표준**

## 고차컴포넌트(HOC)란?

컴포넌트를 인자로 받아 새로운 컴포넌트를 반환하는 함수

``` jsx
const withAuth = (Component) => (props) => { ... }

```
⚠️ 단점
- props 충돌
- 트리 구조 복잡

→ 현재는 **Hooks로 대체**

## React Query(TanStack Query)란? 나온이유와 왜사용하는지

### 무엇인가

- 서버 상태(Server State) 관리 라이브러리

### 왜 나왔나

- API 상태를 Redux로 관리하는 것은 비효율
- 로딩/에러/캐싱/동기화 반복 구현 문제
### 왜 쓰나

- 자동 캐싱
- refetch 전략
- stale 관리
- 에러/로딩 상태 내장


**확실히 더 나은 방법**

> 서버 데이터 → React Query  
> UI 상태 → Local/Global State
