
## Get

목적: 서버로부터 정보를 요청할때

### 특징
1. 데이터를 쿼리스트링을  통해전송
2. HTTP Header에 정보가 담김
3. 브라우저 히스토리에 정보가 남음
4. 길이 제한이 있음
5. 불필요한 요청을 제한하기위해 캐시될수있음


`url에 데이터가 노출되므로 보안취약문제 발생가능`



## Post

목적: 정보를 생성/업데이트하기위해 서버에 데이터를 보내는 HTTP 메소드


## 특징
1. 데이터를 Body에 담아서 전송
2. HTTP Body에 정보가 담김
3. 캐시되지않음
4. 브라우저 히스토리에 남지않음
5. 길이에 제한 X
