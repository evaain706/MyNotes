
## API ROUTE

`백엔드 서버를 따로 두지 않고도 API를 작성할 수 있도록 제공되는 서버 코드`

- 백엔드에서 내려주는 각기 다른 형태의 에러 응답 구조를 API Route 단에서 클라이언트가 처리하기 쉬운 일관된 형태({ message, status } 등)로 정제하여 넘겨줄 수 있어, 프론트엔드 단의 컴포넌트 에러 처리 로직이 훨씬 간결해지는 이점



## Promise.allSettled

`여러 Promise를 병렬로 실행하고, 모든 Promise가 완료될 때까지 기다린 뒤 각각의 결과 상태를 반환하는 메서드`

Promise.all은 하나라도 reject되면 즉시 에러가발생
`성공과 실패를 구분하지않고 모든결과를 반환`
