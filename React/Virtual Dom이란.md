

## Virtual Dom(가상돔)이란?

`React`에서는 가상DOM을 사용하는데 가상돔이란 실제 DOM를 JS객체형태로 메모리에 저장한 형태로  변경전,변경후의 객체를 비교하여 변경될부분만 업데이트하는방식이다.

일반 바닐라 JS에서는 실제 DOM을 조작하여 화면을 다시 그렸지만
예전의 정적이던 사이트에서는 괜찮았지만 최근의 동적인 웹에서는 실제 DOM을 매번 변경하는것이 무거울수도 있어 가상DOM 방식을 채택



이런방식으로 가상DOM은 `Reconciliation(재조정)`과정을 통해
객체 와 객체를 `diffing알고리즘`을 통해 비교하여
변경사항만 실제DOM에 업데이트 한다.
->여러 업데이트가 연달아 있다면 매번 적용하는것이 아닌
   묶어둔뒤 한번에 적용시킴(batching process)

#### 재조정?

**재조정(Reconciliation)** 이란  
**상태(state)나 props가 변경되었을 때**,  
이전 렌더링 결과로 만들어진 Virtual DOM과  
새로 생성된 Virtual DOM을 비교하여
 **실제 DOM에 최소한의 변경만 적용하기 위한 과정**

### 재조정 발생 트리거

- state 변경
- props 변경
- context 값 변경
- 강제 리렌더링

---

## 장단점

### 장점

1. 변경사항만 비교하여 적용하기때문에 실제DOM조작 최소화
2. 실제DOM조작이 최소화 되었기때문에 렌더링 성능 향상
3. 개발자가 직접 DOM조작을 하지 않아도됨
4. 복잡한 UI관리를 효율적으로 가능

### 단점
1. 메모리상에 실제DOM과 완전히 동일한 구조의 메모리를 유지하게되어 추가적인 메모리 사용
2. 간단한 UI나 작은규모의 어플리케이션에서는 불필요한 오버헤드일수도있음
