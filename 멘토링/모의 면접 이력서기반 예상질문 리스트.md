


## 프로젝트 내 문제를 해결하거나 개선하기 위한 구체적 방안을 제시했는가?


- LLM 응답을 JSON으로 강제하도록 시스템 프롬프트를 설계
- 정규식을 활용해 JSON 블록만 추출
- 제어문자 제거 및 파싱 예외 처리 로직 추가

그러나,

- LLM 특성상 간헐적으로 스키마가 깨지는 응답 발생
- 네트워크 중단 또는 토큰 손실로 JSON이 불완전하게 전달되는 케이스 확인


이에 따라:
- 프론트엔드에서 ErrorBoundary를 적용하여 전체 앱 크래시 방지
- fallback UI에서 재요청 CTA 버튼 제공
- 사용자 경험 손실 최소화 및 서비스 안정성 확보

## 프로젝트 중 겪은 갈등과 그 해결 과정, 배운 점을 명확히 설명했는가?

**1. 상황 (Situation) & 문제 (Task)**

• **내용:** 팀 프로젝트(GLOBALNOMAD) 진행 중, 스크럼 회의에서 /로그인 로직 담당 팀원의 개발 일정이 지연된다는 이슈를 공유받았습니다.

• **문제:** 로그인 로직이 완성되어야 예약 인터페이스 등 제 담당 기능의 테스트가 가능한 상황이라, 자칫하면 전체 프로젝트 일정이 밀릴 위기였습니다.

**2. 행동 (Action) - 이력서 내용 반영**

• **기술적 유연성:** "로그인이 완성될 때까지 기다리지 않고, 우선 **Mock Data**와 **Zustand**를 활용해 가상의 로그인 상태를 만들어 제 담당 UI 및 기능 구현을 먼저 진행했습니다."

• **지식 공유:** "동시에 팀원이 어려워하는 부분에 대해 도움을 주었습니다. 제가 이전에 다뤄본 경험이 있는 **Next.js API Route** 설정이나 프록시 통신 부분에 대해 피드백을 제공하여 문제 해결 시간을 단축시켰습니다."

**3. 결과 (Result)**

• **내용:** 결과적으로 제 업무는 유휴 시간(Idle time) 없이 진행되었고, 팀원도 빠르게 로직을 완성하여 **전체 프로젝트 마감 기한(Deadline)을 준수**할 수 있었습니다. 이는 스크럼을 통해 이슈를 조기에 발견하고 대처한 덕분이라고 생각합니다.
