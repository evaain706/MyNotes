
1. 프로젝트 전체 구조 요약
  이 프로젝트는 Next.js (App Router) 기반의 체험 예약 서비스 플랫폼입니다.
   - `src/app`: 라우팅 및 페이지 레이아웃을 담당하며, (with-header)와 (non-header) 그룹을 통해 헤더 노출 여부를 구분합니다.
   - `src/apis`: Axios 인스턴스와 API 호출 함수들을 모듈화하여 관리합니다.
   - `src/components`: 버튼, 입력창, 모달 등 공통 UI 컴포넌트가 위치합니다.
   - `src/hooks`: React Query를 활용한 데이터 페칭 및 공통 비즈니스 로직이 담겨 있습니다.
   - `src/stores`: Zustand를 이용한 전역 상태 관리를 수행합니다.

  ---


  2. src/app/(with-header)/activities/[id] 폴더 분석
  역할: 체험 상세 정보 및 예약 페이지
   * `page.tsx`: ActivityDetailForm을 렌더링하는 진입점입니다.
   * 주요 기능:
       * 정보 전시: 체험의 제목, 카테고리, 설명, 가격, 평점 및 후기 목록을 보여줍니다.
       * 시각 요소: 체험 관련 이미지들을 갤러리 형태로 전시합니다.
       * 위치 정보: LocationMap 컴포넌트를 통해 체험 장소를 지도에 표시합니다.
       * 예약 기능: 사용자가 원하는 날짜와 인원을 선택하여 예약할 수 있는 예약 박스(또는 모달) 기능이 포함되어 있습니다.

  ---


  3. src/app/(with-header)/myactivity 폴더 분석
  역할: 호스트를 위한 체험 등록 및 수정 관리
  이 폴더는 체험을 직접 운영하는 사용자가 자신의 콘텐츠를 관리하는 곳입니다.


   * `myactivity/page.tsx` (내 체험 등록):
       * 새로운 체험을 생성하는 페이지입니다.
       * ReservationForm 컴포넌트를 사용하며, 내부적으로 useCreateActivityForm 훅을 통해 제목, 카테고리, 가격, 주소, 이미지, 예약 가능 시간대 등을 입력받아 서버로 전송합니다.
   * `myactivity/[id]/page.tsx` (내 체험 수정):
       * 이미 등록된 특정 체험의 정보를 수정하는 페이지입니다.
       * EditActivityForm 컴포넌트를 사용하며, 기존 데이터를 불러와 폼에 채워주고 수정한 내용을 반영합니다.
   * 공통 컴포넌트 (`myactivity/components`):
       * 등록과 수정 페이지에서 공통으로 사용하는 InfoSection(기본정보), ScheduleSelectForm(일정설정), ImageSection(이미지업로드) 등이 체계적으로 분리되어 있어 코드 재사용성이 높습니다.

  ---


  요약 및 결론
   * `activities/[id]`는 일반 사용자(게스트)가 체험을 조회하고 예약하는 소비자 중심의 페이지입니다.
   * `myactivity`는 체험 제공자(호스트)가 자신의 상품을 등록하고 수정하는 공급자 중심의 관리 페이지입니다.
  두 폴더 모두 components, hooks, utils를 내부에 별도로 두어 페이지 단위의 캡슐화가 잘 이루어져 있는 구조입니다. 추가로 궁금하신 코드 로직이나 상세 분석이 필요하시면 말씀해 주세요!
