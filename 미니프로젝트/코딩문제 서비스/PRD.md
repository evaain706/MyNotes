# FrontQuiz
  

## 1.  개요

  

### 1.1 프로젝트명

**FrontQuiz** - 프론트엔드 개발자를 위한 퀴즈 학습 플랫폼

  

### 1.2 프로젝트 설명

FrontQuiz는 프론트엔드 개발자들이 JavaScript, React, TypeScript 등 다양한 프론트엔드 기술 스택에 대한 퀴즈를 풀고, 학습 통계를 확인하며, 커뮤니티에서 정보를 공유할 수 있는 웹 애플리케이션입니다.

  



## 2. 목표 및 목적

  

### 2.1 비즈니스 목표

- 프론트엔드 개발자들의 지속적인 학습 동기 부여

- 기술 지식 평가 및 개선 도구 제공

- 개발자 커뮤니티 활성화

  

### 2.2 사용자 목표

- 다양한 프론트엔드 기술에 대한 퀴즈를 통해 지식 검증

- 학습 통계를 통한 자신의 강점/약점 파악

- 오답노트를 통한 체계적인 학습 관리

- 커뮤니티를 통한 지식 공유 및 협업

  

### 2.3 기술 목표

- 모던 웹 기술 스택을 활용한 확장 가능한 아키텍처 구축

- 사용자 경험 최적화 (빠른 로딩, 부드러운 애니메이션)

- 안정적인 인증 및 보안 시스템 구현

  

## 3. 타겟 사용자

  

### 3.1 주요 사용자

- **주니어 프론트엔드 개발자**: 기초 지식 학습 및 검증

- **시니어 프론트엔드 개발자**: 지식 점검 및 커뮤니티 참여

- **프론트엔드 개발 학습자**: 체계적인 학습 관리

  

### 3.2 사용자 페르소나

**페르소나 1: 학습 중인 주니어 개발자**

- 목표: React, TypeScript 등 기술 스택 학습

- 니즈: 체계적인 학습 관리, 오답 복습

- 행동: 정기적으로 퀴즈 풀기, 통계 확인

  

**페르소나 2: 경력 개발자**

- 목표: 지식 점검 및 커뮤니티 참여

- 니즈: 빠른 문제 풀이, 정보 공유

- 행동: 커뮤니티 게시글 작성, 댓글 참여

## 5. 비기능 요구사항

  

### 5.1 성능

- **페이지 로딩 시간**: 초기 로딩 3초 이내

- **API 응답 시간**: 평균 2초 이내

- **이미지 최적화**: WebP 형식 지원, 지연 로딩

- **코드 스플리팅**: 라우트 기반 코드 분할

  

### 5.2 보안

- **인증**: JWT 토큰 기반 인증

- **토큰 갱신**: 자동 토큰 갱신 (401 에러 시)

- **HTTPS**: 모든 통신 암호화

- **XSS 방지**: React 기본 XSS 방지 기능 활용

- **CSRF 방지**: withCredentials 옵션 활용

  

### 5.3 접근성

- **키보드 네비게이션**: 모든 인터랙티브 요소 키보드 접근 가능

- **시맨틱 HTML**: 적절한 HTML 태그 사용

- **반응형 디자인**: 모바일, 태블릿, 데스크톱 지원

  

### 5.4 호환성

- **브라우저**: Chrome, Firefox, Safari, Edge 최신 2개 버전

- **모바일**: iOS Safari, Chrome Mobile

- **해상도**: 320px 이상 지원

  

### 5.5 확장성

- **모듈화**: 기능별 모듈 분리 (features 폴더)

- **재사용성**: 공통 컴포넌트 라이브러리

- **타입 안정성**: TypeScript 전면 적용

  

## 6. 사용자 스토리

  

### 6.1 인증 관련

- **US-1**: 사용자는 카카오 계정으로 로그인할 수 있다.

- **US-2**: 사용자는 로그아웃할 수 있다.

- **US-3**: 사용자는 로그인하지 않으면 마이페이지에 접근할 수 없다.

  

### 6.2 퀴즈 관련

- **US-4**: 사용자는 원하는 카테고리를 선택할 수 있다.

- **US-5**: 사용자는 난이도를 선택할 수 있다.

- **US-6**: 사용자는 선택한 옵션에 맞는 퀴즈를 받을 수 있다.

- **US-7**: 사용자는 퀴즈를 풀고 채점받을 수 있다.

- **US-8**: 사용자는 틀린 문제를 오답노트에 저장할 수 있다.

  

### 6.3 통계 관련

- **US-9**: 사용자는 자신의 학습 통계를 확인할 수 있다.

- **US-10**: 사용자는 카테고리별 통계를 확인할 수 있다.

- **US-11**: 사용자는 난이도별 통계를 확인할 수 있다.

  

### 6.4 오답노트 관련

- **US-12**: 사용자는 저장된 오답 목록을 볼 수 있다.

- **US-13**: 사용자는 오답을 필터링할 수 있다 (카테고리, 난이도).

- **US-14**: 사용자는 오답을 삭제할 수 있다.

- **US-15**: 사용자는 오답을 다시 풀 수 있다.

  

### 6.5 커뮤니티 관련

- **US-16**: 사용자는 게시글 목록을 볼 수 있다.

- **US-17**: 사용자는 게시글을 검색할 수 있다.

- **US-18**: 사용자는 카테고리별로 게시글을 필터링할 수 있다.

- **US-19**: 사용자는 게시글을 작성할 수 있다.

- **US-20**: 사용자는 게시글을 읽을 수 있다.

- **US-21**: 사용자는 게시글에 댓글을 작성할 수 있다.

- **US-22**: 사용자는 자신이 작성한 게시글을 수정/삭제할 수 있다.

  

## 7. 기술 요구사항

  

### 7.1 프론트엔드 기술 스택

- **프레임워크**: React 19

- **언어**: TypeScript 5.9

- **빌드 도구**: Vite 7

- **라우팅**: React Router v7

- **상태 관리**: Zustand 5, TanStack React Query v5

- **HTTP 클라이언트**: Axios 1.12

- **스타일링**: Tailwind CSS v4

- **애니메이션**: Motion 12

- **폼 관리**: React Hook Form 7

- **마크다운**: React Markdown 10

  

### 7.2 아키텍처 패턴

- **Feature-based 구조**: 기능별 모듈 분리

- **컴포넌트 조합 패턴**: Modal, Dropdown 등 Context 기반 컴포넌트

- **커스텀 훅 패턴**: 비즈니스 로직 분리

- **스토어 패턴**: Zustand를 통한 전역 상태 관리

  

### 7.3 API 설계

- **에러 처리**: 표준화된 에러 응답 형식

## 10. 제약사항 및 가정

  

### 10.1 제약사항

- **백엔드 API 의존성**: 백엔드 API가 정상 작동해야 함

- **카카오 OAuth**: 카카오 개발자 앱 등록 필요

- **브라우저 호환성**: 최신 브라우저 필요

- **네트워크**: 인터넷 연결 필수

  

### 10.2 가정

- 사용자는 기본적인 웹 브라우저 사용 능력이 있음

- 사용자는 카카오 계정을 보유하고 있음

- 백엔드 API는 안정적으로 운영됨

- 사용자는 프론트엔드 개발에 대한 기본 지식이 있음
